HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML attributes are always defined in the beginning element, never the closing element
  • Important metadata tags within the 'head' element
    • The 'title' element defines the title for a webpage, it is required and imporant for SEO
    • The 'style' element defines the CSS styles associated with a website
    • The 'base' element defines the base URL for a webpage
    • The 'link' element connects an external resource to the HTML document
    • The 'meta' element defines metadata such as the character set, description, keywords, author, and viewport
    • The 'script' element embeds client-side scripts in the HTML document

CSS

  • CSS rules have a selector followed by a declaration block denoted within curly brackets '{}'
    • A selector is what we want the CSS code to effect
    • The declaration block is where we write how we would like the CSS code to effect the selector
      • Within the declaration block, for each declaration, there is a property and corresponding property value, separated by a colon ':'
      • Within each ruleset you must use a semicolon ';' to separate each declaration from the next
  • A margin indicates how much space we want around thte outside of an element.
  • A padding indicates how much space we want around the content inside an element.

GIT

  • The following list describes the order of commands to create a branch
    • Navigate to the correct directory
    • Type 'git status' to check which branch you're currently in
    • If you are not in the 'main' branch enter the command 'git checkout main'
    • Pull down the latest main branch by entering 'git pull origin main'
    • Enter the command 'git checkout -b feature/new-branch-name' to create a new branch
    • Confirm you are on the new branch by entering the command 'git status'
  • The following list describes the order of commands to merge the working branch to the main branch, aka 'completing the git flow'
    • Be sure you are on the correct working branch, and then use the command 'git add -A' to commit ALL files in the working branch
    • Now that all of the files have been staged, commit them to the repo using the command 'git commit -m "description of changes"'
    • It is good habit to always pull the base branch into your feature branch before pushing up any code; enter 'git pull origin main' to do so
    • Next, push up all of your changes with the command 'git push origin feature/new-branch-name'
    • You will then navigate in GitHub to the corresponding repository and do the following to merge the working branch to the main
      • Change the branch to the 'feature/new-branch-name' and refresh the page to confirm the changes pushed are in the feature branch
      • Navigate to the 'Pull Requests' tab and open a new pull request; make sure you select 'main' as the base and 'feature' as the comparison branch
      • Click 'Create pull request' to open a PR
      • Give the PR a title and short description and then click 'Merge pull request' to merge
        • *Remember to have someone review your work before merging
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • The 'touch' command following by a file name creates a file in your current directory

JAVASCRIPT

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • APIs are browser application programming interfaces built into web browsers, providing functionality
  • Comments are snippets of text that can be added along with code by starting the comment with '/*' and ending it with '*/'
    • If your comment contains no line breaks, it's an option to put it behind two slashes ' // '
  • An 'operator' is a mathmatical symbol that produces a result based on two vaules (or variables)
  • 'Conditionals' are code structures used to test if an expression returns true or not like the 'if' and 'else' statement
